Detailed Description

template<class T>
class xl::magic_ptr< T >

A magic_ptr is similar to a unique_ptr but it knows if it's holding a "reference" or an owning pointer based on whether it was created with an rvalue or lvalue reference. Both kinds of magic_ptr's have the same type, however, so they can be used interchangeably in a homogenous container magic_ptr's are move-only like a unique_ptr, as they may own memory. Note: magic_ptr's are HUGE, likely 5x a normal pointer. Use with care.
